## 1. โจทย์ของทีม

**Problem statement** (1-2 ประโยค บอกว่าจะแก้ปัญหาอะไรในชีวิตจริง):

```
ปัญหาการคัดแยกชนิดของมะพร้าวส่งผลกระทบต่อทุกภาคส่วนที่เกี่ยวข้องในห่วงโซ่อุปทาน ตั้งแต่เกษตรกร ผู้ประกอบการ แรงงาน ผู้บริโภค ไปจนถึงภาคอุตสาหกรรมและเศรษฐกิจของประเทศ ดังนั้น การพัฒนาระบบคัดแยกที่มีประสิทธิภาพและแม่นยำจึงมีความสำคัญต่อการเพิ่มคุณภาพสินค้า ลดต้นทุน และยกระดับมาตรฐานอุตสาหกรรมมะพร้าวในอนาคต

```

## 2. Class Definition

ระบุ class ที่จะใช้ + เกณฑ์ที่ทำให้แต่ละ class แตกต่าง:

| # | ชื่อ Class | คำอธิบาย (3-4 ประโยค) | สิ่งที่ทำให้ต่างจาก class อื่น |
|---|---|---|---|
| 1 | CO | มะพร้าวอ่อน | มีสีส้ม |
| 2 | CM | กะทิ | มีสีฟ้า |
| 3 | BG | พื้นหลัง | ทุกอย่างที่ไม่ใช้ |

**ตัวอย่าง (Track A, Fall Detector):**

| # | ชื่อ Class | คำอธิบาย | สิ่งที่ทำให้ต่าง |
|---|---|---|---|
| 1 | ยืน | ผู้ใช้ยืนนิ่ง หรือยืนเฉยๆ ใส่ที่เอว | accel แกน Z สูง, gyro ต่ำ |
| 2 | เดิน | เดินปกติ ก้าวเท้าสลับ | accel oscillation, periodic |
| 3 | ล้ม | ล้มลงพื้นกะทันหัน | high accel spike + horizontal Z |

## 3. Edge Cases — สิ่งที่อาจสับสน

ระบุสถานการณ์ที่อาจทำให้โมเดลสับสน:

```
1. ระดับแสงที่เข้มหรือจางเกิน จะทำให้คาดเดาคลาดเคลื่อนได้

2. ระยะของมุมกล้องที่ไม่คงที่ ทำให้คาดเดาไม่ได้หรือผืดพลาด

3. มีวัตถุที่มีสีที่คลายกับ CO & CM จะทำให้เดาเป็น CO หรือ CM ได้
```

**วางแผนแก้:**
```
1. ใช้พื้นที่ที่มีแสงไม่มากเกินไปหรือไม่มืดจนวัดไม่ได้
2. สร้าง Prototype ที่มีความสูงคงที่ในการ Train
3. Train AI ด้วยภาพ Sample ที่มากขึ้น
```

## 5. Diversity Plan — กันอคติ (Bias)

ตอบ 3 คำถาม:

### a) มีหลาย "environment" ไหม?
```
[เก็บในห้องเรียน แสงสว่าง เวลาไรเลี้ยกัน]
```

### b) มีหลาย "subject" ไหม? (กรณี Vision/Audio/Motion)
```
[เก็บโดย ลักษณะ background ต่างกัน?]
```

### c) มีหลาย "variation" ไหม?
```
[มุม/ระยะ/ความเร็ว/ระดับเสียง ที่แตกต่างกัน?]
```
